Use this path for **AWS account-level observation** — monitor your AWS infrastructure with EC2, RDS, S3, Lambda, and more.

<Tip>
⏱️ **Expected time:** 5-10 minutes
</Tip>

---

## 1) Prerequisites

<AccordionGroup>
  <Accordion title="RubixKube Account" icon="user">
    - Visit [console.rubixkube.ai](https://console.rubixkube.ai)
    - [Sign up](/getting-started/sign-up) or [log in](/getting-started/login)
    - Copy your API key from the dashboard
  </Accordion>

  <Accordion title="AWS CLI" icon="aws">
    - Install AWS CLI v2 from [aws.amazon.com/cli](https://aws.amazon.com/cli/)
    - Run `aws configure` with your credentials
    - Verify: `aws sts get-caller-identity`
  </Accordion>

  <Accordion title="IAM Permissions" icon="shield">
    Your AWS credentials need read-only permissions for:
    - EC2, RDS, S3, Lambda, CloudWatch, CloudTrail, ELB

    The installer automatically creates the required IAM policy and role.
  </Accordion>
</AccordionGroup>

---

## 2) Run the Installer

```bash
curl -fsSL https://api.rubixkube.ai/install/observer.sh | bash -s -- --api-key=rk_YOUR_API_KEY
```

<Warning>
Use your personal command from [console.rubixkube.ai](https://console.rubixkube.ai) — your API key is embedded in it.
</Warning>

---

## 3) Follow the Interactive Flow

When prompted:

```
What would you like to do?
  1) Install Observer
  2) Uninstall Observer
> 1
```

Then:

```
Where do you want to deploy the observer?
  1) On this machine
  2) Create a new AWS EC2 instance
  3) Create a new GCP Compute Engine instance
> 2  (or 1 for existing VM)
```

Then:

```
AWS Region [us-east-1]:
```

Enter your desired region (default: `us-east-1`).

---

## 4) Verify Installation

### Option A: New EC2 Instance

Check your EC2 console:
```bash
# View running observers
aws ec2 describe-instances --filters "Name=tag:rubixkube,Values=observer" \
  --query 'Reservations[].Instances[].{ID:InstanceId,Status:State.Name,Name:Tags[?Key==`Name`]|[0].Value}'
```

Expected output:
```
ID              Status   Name
i-0abc123...    running  rk-observer-20250101120000
```

### Option B: Existing Machine

Check service status:
```bash
sudo systemctl status rubixkube-observer
```

View logs:
```bash
sudo journalctl -u rubixkube-observer -f
```

---

## 5) Verify in RubixKube Dashboard

1. Open [console.rubixkube.ai](https://console.rubixkube.ai)
2. Go to **Environments** or **Infrastructure**
3. Your AWS environment should appear (labeled with account ID)
4. Wait 1-2 minutes for metrics to populate

## Troubleshooting

<AccordionGroup>
  <Accordion title="Observer won't start">
    Check for errors:
    ```bash
    # For EC2 instance
    aws ec2 get-console-output --instance-id i-xxxxxx --query 'Output' | grep -i error

    # For existing machine
    sudo journalctl -u rubixkube-observer -n 50
    ```

    Common issues: Invalid API key, missing AWS credentials, firewall blocking 443 to `api.rubixkube.ai`
  </Accordion>

  <Accordion title="Metrics not appearing">
    1. Verify observer is running (see above)
    2. Wait 2-3 minutes for first metrics
    3. Check AWS API access: `aws ec2 describe-instances`
    4. Verify environment in dashboard
  </Accordion>

  <Accordion title="Permission denied errors">
    Ensure AWS credentials have read-only access to:
    ```
    ec2:Describe*, rds:Describe*, s3:List*, lambda:List*, cloudwatch:*
    ```

    Run: `aws iam get-user` to verify credentials
  </Accordion>
</AccordionGroup>

## What's Being Monitored?

Your AWS observer is now collecting:
- ✅ EC2 instance metrics and status
- ✅ RDS database performance and connections
- ✅ S3 bucket inventory and access patterns
- ✅ Lambda function invocations and errors
- ✅ Load balancer traffic and health
- ✅ CloudTrail API activity and events
