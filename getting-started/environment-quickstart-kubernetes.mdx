Use this path for **Kubernetes environments** (EKS, GKE, AKS, or local KIND).

<Tip>
⏱️ **Expected time:** 5-10 minutes
</Tip>

---

## 1) Prerequisites

<AccordionGroup>
  <Accordion title="RubixKube Account" icon="user">
    - Visit [console.rubixkube.ai](https://console.rubixkube.ai)
    - [Sign up](/getting-started/sign-up) or [log in](/getting-started/login)
    - Copy your API key from the dashboard
  </Accordion>

  <Accordion title="kubectl CLI" icon="terminal">
    - Install `kubectl` from [kubernetes.io/docs/tasks/tools](https://kubernetes.io/docs/tasks/tools/)
    - Verify: `kubectl version --client`
    - Configure access to your cluster: `kubectl get nodes`
  </Accordion>

  <Accordion title="Kubernetes Cluster" icon="cubes">
    - EKS, GKE, AKS, or local KIND
    - Kubernetes v1.24+
    - Access to the cluster with admin permissions
  </Accordion>
</AccordionGroup>

---

## 2) Get Your Installation Command

1. Log in to [console.rubixkube.ai](https://console.rubixkube.ai)
2. Copy the **kubectl installation command** from your dashboard
3. It looks like: `kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_API_KEY"`

<Warning>
Use your personal command from the dashboard — your API key is embedded in it.
</Warning>

---

## 3) Run the Installation

```bash
kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_API_KEY"
```

You should see:
```
namespace/rubixkube-system created
serviceaccount/rubixkube-observer created
clusterrole.rbac.authorization.k8s.io/rubixkube-observer created
deployment.apps/rubixkube-observer created
```

---

## 4) Verify Installation

Check that the observer pod is running:

```bash
kubectl get pods -n rubixkube-system
```

Wait for **STATUS** to show `Running`:

```
NAME                                  READY   STATUS    RESTARTS   AGE
rubixkube-observer-7d4b9c8f6d-abc12   1/1     Running   0          2m
```

View logs:
```bash
kubectl logs -n rubixkube-system deployment/rubixkube-observer --tail=20
```

---

## 5) Verify in RubixKube Dashboard

1. Open [console.rubixkube.ai](https://console.rubixkube.ai)
2. Go to **Environments** or **Infrastructure**
3. Your Kubernetes environment should appear
4. Wait 1-2 minutes for metrics to populate

## Troubleshooting

<AccordionGroup>
  <Accordion title="Pod stuck in ImagePullBackOff">
    **Problem:** Pod can't download image

    **Solution:** Wait 2-3 minutes, or check cluster internet access:
    ```bash
    kubectl run test-curl --image=curlimages/curl -i --rm -- curl -I https://api.rubixkube.ai
    ```
  </Accordion>

  <Accordion title="Pod in CrashLoopBackOff">
    **Problem:** Pod crashes immediately

    **Check logs:**
    ```bash
    kubectl logs -n rubixkube-system deployment/rubixkube-observer --previous
    ```

    **Most common cause:** Invalid API key. Reinstall with correct key from dashboard.
  </Accordion>

  <Accordion title="Environment not appearing in dashboard">
    **Checklist:**
    1. Wait 2-3 minutes for sync
    2. Verify pod is running: `kubectl get pods -n rubixkube-system`
    3. Check logs for connection errors
    4. Verify cluster has outbound access to `api.rubixkube.ai:443`
  </Accordion>
</AccordionGroup>

## What's Being Monitored?

Your Kubernetes observer is now collecting:
- ✅ Pod metrics and status (CPU, memory, network)
- ✅ Deployment rollouts and replica status
- ✅ Service discovery and networking
- ✅ Node metrics and capacity
- ✅ Cluster events and state changes
