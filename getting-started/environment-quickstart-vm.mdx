Use this path for **generic Linux VM monitoring** — monitor CPU, memory, disk, network, and process-level metrics on any Linux host.

<Tip>
⏱️ **Expected time:** 5 minutes
</Tip>

---

## 1) Prerequisites

<AccordionGroup>
  <Accordion title="RubixKube Account" icon="user">
    - Visit [console.rubixkube.ai](https://console.rubixkube.ai)
    - [Sign up](/getting-started/sign-up) or [log in](/getting-started/login)
    - Copy your API key from the dashboard
  </Accordion>

  <Accordion title="Linux VM" icon="server">
    - Linux host (Amazon Linux, Ubuntu, Debian, RHEL, CentOS, Alpine)
    - Outbound access to `api.rubixkube.ai:443`
    - Sudo or root access for service installation
  </Accordion>

  <Accordion title="Basic Tools" icon="terminal">
    - `curl` or `wget` (for downloading)
    - `bash` shell
    - `systemd` for service management
  </Accordion>
</AccordionGroup>

---

## 2) Run the Installer

On your Linux VM:

```bash
curl -fsSL https://api.rubixkube.ai/install/observer.sh | bash -s -- --api-key=rk_YOUR_API_KEY
```

<Warning>
Use your personal command from [console.rubixkube.ai](https://console.rubixkube.ai) — your API key is embedded in it.
</Warning>

---

## 3) Follow the Interactive Flow

When prompted:

```
What would you like to do?
  1) Install Observer
  2) Uninstall Observer
> 1
```

Then:

```
Where do you want to deploy the observer?
  1) On this machine
  2) Create a new AWS EC2 instance
  3) Create a new GCP Compute Engine instance
> 1
```

Then:

```
Which platform are you monitoring?
  1) VM (generic Linux)
  2) AWS
  3) GCP
> 1
```

---

## 4) Verify Installation

Check service status:
```bash
sudo systemctl status rubixkube-observer
```

Expected output:
```
● rubixkube-observer.service - RubixKube Observer
   Loaded: loaded
   Active: active (running)
```

View logs:
```bash
sudo journalctl -u rubixkube-observer -f
```

Look for:
```
[INFO] RubixKube Observer started
[INFO] VM mode enabled
[INFO] System polling enabled
[INFO] Publishing metrics to NATS
```

---

## 5) Verify in RubixKube Dashboard

1. Open [console.rubixkube.ai](https://console.rubixkube.ai)
2. Go to **Environments** or **Infrastructure**
3. Your VM should appear (labeled with hostname)
4. Wait 1-2 minutes for metrics to populate

## Troubleshooting

<AccordionGroup>
  <Accordion title="Service won't start">
    Check for errors:
    ```bash
    sudo journalctl -u rubixkube-observer -n 50
    ```

    Common issues: Invalid API key, permission denied, port already in use
  </Accordion>

  <Accordion title="Metrics not appearing">
    1. Verify service is running: `sudo systemctl status rubixkube-observer`
    2. Wait 2-3 minutes for first metrics
    3. Check connectivity: `curl -I https://api.rubixkube.ai`
    4. Verify environment in dashboard
  </Accordion>

  <Accordion title="High CPU/Memory usage">
    This is rare, but try:
    ```bash
    # Increase polling interval (default: 30s)
    sudo bash -c 'echo "SYSTEM_POLL_INTERVAL=60s" >> /etc/rubixkube/observer.env'
    sudo systemctl restart rubixkube-observer
    ```
  </Accordion>

  <Accordion title="Firewall blocking outbound traffic">
    Ensure outbound TCP 443 is allowed to:
    ```
    api.rubixkube.ai
    nats.rubixkube.ai
    ```

    Test connectivity: `curl -I https://api.rubixkube.ai`
  </Accordion>
</AccordionGroup>

## What's Being Monitored?

Your VM observer is now collecting:
- ✅ CPU usage, load averages, per-core metrics
- ✅ Memory usage, swap, available/cached
- ✅ Disk usage per mount, free space, I/O metrics
- ✅ Network interface stats, errors, drops
- ✅ Process metrics, top consumers, zombie detection
- ✅ System events and alerts
