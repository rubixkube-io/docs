Welcome! This guide will help you install RubixKube on your **local machine** using KIND (Kubernetes in Docker). This is perfect for:

- **Testing** RubixKube before deploying to production
- **Learning** how RubixKube works in a safe environment
- **Development** on your laptop without cloud costs
- **Training** team members on RubixKube features

<Info>
<strong>Complete Beginner?</strong> No problem! We'll walk you through installing everything from scratch, even if you've never used Docker or Kubernetes before.
</Info>

## Step 1: Install Docker

Docker is required to run KIND. Choose your operating system below:

<Tabs>
  <Tab title="macOS">
    ### Install Docker Desktop on macOS

### Option 1: Using Homebrew (Recommended)
    If you have Homebrew installed:

    ```bash
    brew install --cask docker
    ```

### Option 2: Download from Docker Website
    1. Visit [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop)
    2. Click **"Download for Mac"**
    3. Choose your Mac type:
       - **Apple Silicon** (M1/M2/M3 Macs)
       - **Intel Chip** (older Macs)
    4. Open the downloaded `.dmg` file
    5. Drag Docker icon to Applications folder
    6. Launch Docker from Applications

    ### Verify Docker Installation

    Open Terminal and run:

    ```bash
    docker --version
    ```

    You should see output like:
    ```
    Docker version 24.0.6, build ed223bc
    ```

    <Tip>
<strong>First Time?</strong> Docker may ask for system permissions. Click "OK" to allow it.
    </Tip>
  </Tab>

  <Tab title="Windows">
    ### Install Docker Desktop on Windows

**Prerequisites:** - Windows 10 64-bit: Pro, Enterprise, or Education (Build 19041 or higher)
    - Windows 11 64-bit
    - WSL 2 (Windows Subsystem for Linux)

### Step 1: Enable WSL 2
    Open PowerShell as Administrator and run:

    ```powershell
    wsl --install
    ```

    Restart your computer after this completes.

### Step 2: Download Docker Desktop
    1. Visit [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop)
    2. Click **"Download for Windows"**
    3. Run the installer (`Docker Desktop Installer.exe`)
    4. Follow the installation wizard
    5. Ensure **"Use WSL 2 instead of Hyper-V"** is checked
    6. Complete installation and restart if prompted

### Step 3: Launch Docker Desktop
    1. Start Docker Desktop from Start menu
    2. Accept the service agreement
    3. Wait for Docker Engine to start (whale icon in system tray)

    ### Verify Docker Installation

    Open PowerShell or Command Prompt and run:

    ```powershell
    docker --version
    ```

    You should see output like:
    ```
    Docker version 24.0.6, build ed223bc
    ```

    <Warning>
<strong>Virtualization Required:</strong> Docker Desktop requires hardware virtualization. Enable VT-x/AMD-V in your BIOS if you see errors.
    </Warning>
  </Tab>

  <Tab title="Linux">
    ### Install Docker on Linux

### For Ubuntu/Debian:
    ```bash
    # Update package index
    sudo apt-get update

    # Install prerequisites
    sudo apt-get install -y \
        ca-certificates \
        curl \
        gnupg \
        lsb-release

    # Add Docker's official GPG key
    sudo mkdir -p /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
        sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

    # Set up the repository
    echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
      https://download.docker.com/linux/ubuntu \
      $(lsb_release -cs) stable" | \
      sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    # Install Docker Engine
    sudo apt-get update
    sudo apt-get install -y docker-ce docker-ce-cli containerd.io

    # Add your user to docker group (avoid using sudo)
    sudo usermod -aG docker $USER

    # Log out and back in for group changes to take effect
    ```

### For Fedora/RHEL/CentOS:
    ```bash
    # Install Docker
    sudo dnf -y install dnf-plugins-core
    sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
    sudo dnf install -y docker-ce docker-ce-cli containerd.io

    # Start Docker service
    sudo systemctl start docker
    sudo systemctl enable docker

    # Add your user to docker group
    sudo usermod -aG docker $USER
    ```

    ### Verify Docker Installation

    Log out and log back in, then run:

    ```bash
    docker --version
    ```

    You should see output like:
    ```
    Docker version 24.0.6, build ed223bc
    ```

    <Tip>
<strong>Permission Denied?</strong> Make sure you logged out and back in after adding your user to the docker group.
    </Tip>
  </Tab>
</Tabs>

## Step 2: Install KIND

KIND (Kubernetes in Docker) creates Kubernetes clusters using Docker containers as nodes.

<Tabs>
  <Tab title="macOS">
    ### Install KIND on macOS

### Using Homebrew (Recommended):
    ```bash
    brew install kind
    ```

### Manual Installation:
    ```bash
    # Download KIND binary
    curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-darwin-amd64

    # Make it executable
    chmod +x ./kind

    # Move to system path
    sudo mv ./kind /usr/local/bin/kind
    ```

    ### Verify KIND Installation

    ```bash
    kind --version
    ```

    Expected output:
    ```
    kind version 0.20.0
    ```
  </Tab>

  <Tab title="Windows">
    ### Install KIND on Windows

### Using Chocolatey (Recommended):
    If you have Chocolatey installed:

    ```powershell
    choco install kind
    ```

### Manual Installation:
    1. Download KIND from [GitHub Releases](https://github.com/kubernetes-sigs/kind/releases)
    2. Choose `kind-windows-amd64`
    3. Rename it to `kind.exe`
    4. Move it to a directory in your PATH (e.g., `C:\Windows\System32`)

### Or using PowerShell:
    ```powershell
    curl.exe -Lo kind-windows-amd64.exe https://kind.sigs.k8s.io/dl/v0.20.0/kind-windows-amd64
    Move-Item .\kind-windows-amd64.exe C:\Windows\System32\kind.exe
    ```

    ### Verify KIND Installation

    ```powershell
    kind --version
    ```

    Expected output:
    ```
    kind version 0.20.0
    ```
  </Tab>

  <Tab title="Linux">
    ### Install KIND on Linux

    ```bash
    # Download KIND binary
    curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-linux-amd64

    # Make it executable
    chmod +x ./kind

    # Move to system path
    sudo mv ./kind /usr/local/bin/kind
    ```

    ### Verify KIND Installation

    ```bash
    kind --version
    ```

    Expected output:
    ```
    kind version 0.20.0
    ```
  </Tab>
</Tabs>

## Step 4: Create a KIND Cluster

Now that we have all prerequisites installed, let's create a local Kubernetes cluster!

### Create Your Cluster

Run this command in your terminal:

```bash
kind create cluster --name rubixkube-local
```

**What's happening?** - KIND creates a Kubernetes cluster running inside Docker containers
- The cluster is named `rubixkube-local`
- This takes 1-2 minutes

### Expected Output:
```
Creating cluster "rubixkube-local" ...
  Ensuring node image (kindest/node:v1.27.3) 
  Preparing nodes 
  Writing configuration 
  Starting control-plane 
  Installing CNI 
  Installing StorageClass 
Set kubectl context to "kind-rubixkube-local"
You can now use your cluster with:

kubectl cluster-info --context kind-rubixkube-local
```

    <Tip>
<strong>Pro Tip:</strong> The cluster automatically sets itself as your active Kubernetes context, so kubectl commands will target it.
    </Tip>

### Verify Your Cluster is Running

```bash
kubectl cluster-info
```

Expected output:
```
Kubernetes control plane is running at https://127.0.0.1:xxxxx
CoreDNS is running at https://127.0.0.1:xxxxx/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy
```

Check that nodes are ready:

```bash
kubectl get nodes
```

Expected output:
```
NAME                            STATUS   ROLES           AGE   VERSION
rubixkube-local-control-plane   Ready    control-plane   1m    v1.27.3
```

    <Note>
<strong>Troubleshooting:</strong> If the node shows "NotReady", wait 30 seconds and check again. It takes time to initialize.
    </Note>

## Step 5: Install RubixKube Observer

Now let's install the RubixKube observer agent on your local cluster!

### Get Your Installation Command

1. **Log in to RubixKube Console** at [console.rubixkube.ai](https://console.rubixkube.ai)
2. After login, you'll see the **"Connect Your First Cluster"** onboarding screen
3. **Copy the kubectl command** shown on the page

It will look something like this:

```bash
kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_API_KEY_HERE"
```

<strong>Important:</strong> Use YOUR specific command from the dashboard - it contains your unique API key!

Paste your command into the terminal and press Enter:

```bash
kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_API_KEY_HERE"
```

### Expected Output:
```

serviceaccount/rubixkube-observer created
clusterrole.rbac.authorization.k8s.io/rubixkube-observer created

    <Tip>
<strong>What just happened?</strong> RubixKube installed an "observer" agent in your cluster that monitors your infrastructure and reports back to the RubixKube platform.

## Step 6: Verify RubixKube Installation

Let's make sure RubixKube is running correctly in your cluster!

### Check if Pods are Running

```bash
kubectl get pods -n rubixkube
```

**Initial output** (pods are starting):
```
NAME                                  READY   STATUS              RESTARTS   AGE
rubixkube-observer-7d4b9c8f6d-x7k2p   0/1     ContainerCreating   0          10s
```

**Wait 30-60 seconds**, then run the command again:

```bash
kubectl get pods -n rubixkube
```

**Success!** You should see:
```
NAME                                  READY   STATUS    RESTARTS   AGE
rubixkube-observer-7d4b9c8f6d-x7k2p   1/1     Running   0          1m
```

<AccordionGroup>
  <Accordion title=" What does successful installation look like?">
**READY column shows:** `1/1` (1 out of 1 containers ready)  
**STATUS column shows:** `Running`  
**RESTARTS column shows:** `0` (or low number)

    This means RubixKube observer is:
    - Successfully deployed
    - Running and healthy
    - Connected to RubixKube cloud
    - Monitoring your cluster
  </Accordion>

  <Accordion title=" What if STATUS shows 'ImagePullBackOff'?">
    This means Docker couldn't download the RubixKube image.

**Common causes:** - Internet connection issues
    - Docker Hub rate limits

**Solution:** Wait 2-3 minutes and check again:
    ```bash
    kubectl get pods -n rubixkube
    ```

    If still failing after 5 minutes, delete and reinstall:
    ```bash
    kubectl delete namespace rubixkube
    # Wait 30 seconds, then run the install command again
    ```
  </Accordion>

  <Accordion title=" What if STATUS shows 'CrashLoopBackOff'?">
    This means the pod started but crashed.

**Check logs to see why:** ```bash
    kubectl logs -n rubixkube deployment/rubixkube-observer
    ```

**Common causes:**    - Invalid API key
    - Configuration error

**Solution:**    1. Double-check you used YOUR api key from the dashboard
    2. Delete and reinstall with correct command:
    ```bash
    kubectl delete namespace rubixkube
    # Get fresh command from dashboard and run it
    ```
  </Accordion>

  <Accordion title=" How do I see detailed pod information?">
    Get detailed information about the pod:
    ```bash
    kubectl describe pod -n rubixkube -l app=rubixkube-observer
    ```

    This shows:
    - Events (what happened during startup)
    - Container status
    - Resource usage
    - Error messages (if any)
  </Accordion>
</AccordionGroup>

### Check Observer Logs

See what RubixKube observer is doing:

```bash
kubectl logs -n rubixkube deployment/rubixkube-observer
```

**Healthy logs look like:**
```text
[INFO] RubixKube Observer starting...
[INFO] Connected to RubixKube API
[INFO] Cluster registration successful
[INFO] Starting infrastructure monitoring
[INFO] Watching namespaces: all
[INFO] Agent mesh initialized
```

<Tip>
<strong>Tip:</strong> If you see steady log output with [INFO] messages, everything is working correctly!
</Tip>

### Verify in RubixKube Dashboard

1. Go back to [console.rubixkube.ai](https://console.rubixkube.ai)
2. You should see your cluster appear in the dashboard
3. The **Observer Status** should change from "Awaiting installation" to **"Connected"**

<Note>
<strong>Note:</strong> It may take 1-2 minutes for your cluster to appear in the dashboard after the pod starts running.
</Note>

## Managing Your KIND Cluster

### Useful Commands

**List all KIND clusters:** 
```bash
kind get clusters
```

**Stop your cluster** (cluster persists, just stops):
```bash
docker stop rubixkube-local-control-plane
```

**Start your cluster again:** ```bash
docker start rubixkube-local-control-plane
```

**Delete your cluster** (removes everything):
```bash
kind delete cluster --name rubixkube-local
```

**Create cluster with custom configuration:**

```bash
cat > kind-config.yaml <<'EOF'
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
- role: worker
- role: worker
EOF

kind create cluster --name rubixkube-local --config=kind-config.yaml

# Clean up config file
rm kind-config.yaml
```

This creates a cluster with 1 control plane and 2 worker nodes.

## Common Issues & Solutions

<AccordionGroup>
  <Accordion title="Docker daemon not running">
**Error:** `Cannot connect to the Docker daemon`

**Solution:**    - **macOS/Windows:** Open Docker Desktop app and wait for it to start
    - **Linux:** Run `sudo systemctl start docker`
  </Accordion>

  <Accordion title="KIND cluster creation fails">
**Error:** `failed to create cluster`

**Solutions:**    1. Ensure Docker has enough resources (4GB+ RAM recommended)
    2. Check Docker is running: `docker ps`
    3. Delete any existing cluster: `kind delete cluster --name rubixkube-local`
    4. Try creating again
  </Accordion>

  <Accordion title="kubectl: command not found">
**Error:** `kubectl: command not found`

**Solution:**    - Kubectl is not in your system PATH
    - Re-install kubectl and ensure it's added to PATH
    - Try full path: `/usr/local/bin/kubectl` (macOS/Linux)
  </Accordion>

  <Accordion title="Port already in use">
**Error:** `address already in use`

**Solution:**    Another process is using Kubernetes default ports.
    
    Delete existing KIND clusters:
    ```bash
    kind delete cluster --name rubixkube-local
    ```
    
    Or specify custom ports when creating cluster.
  </Accordion>

  <Accordion title="RubixKube observer not starting">
### Check these:
    1. **Verify API key is correct**
    ```bash
    kubectl get secret -n rubixkube rubixkube-credentials -o yaml
    ```

    2. **Check pod events**
    ```bash
    kubectl describe pod -n rubixkube -l app=rubixkube-observer
    ```

    3. **View logs**
    ```bash
    kubectl logs -n rubixkube deployment/rubixkube-observer
    ```

    4. **Reinstall**
    ```bash
    kubectl delete namespace rubixkube
    # Wait 30 seconds, then run install command again
    ```
  </Accordion>
</AccordionGroup>

## Comparison: KIND vs Cloud

| Feature | KIND (Local) | Cloud Cluster |
|---------|-------------|---------------|
| **Cost**  | Free | Paid |
| **Setup Time**  | 5-10 minutes | 10-30 minutes |
| **Use Case**  | Testing, Learning | Production |
| **Performance**  | Limited by your machine | Full cloud resources |
| **Persistence**  | Data lost when cluster deleted | Persistent storage |
| **Internet Access**  | Works offline (after setup) | Always requires internet |
| **Load Balancers**  | Simulated only | Real cloud load balancers |

<Tip>
<strong>Best Practice:</strong> Start with KIND to learn RubixKube, then move to cloud for production workloads.
</Tip>
