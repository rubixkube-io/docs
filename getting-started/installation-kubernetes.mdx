Install RubixKube on your **Kubernetes cluster** to monitor workload-level infrastructure and get pod-level reliability insights.

<Info>
Need AWS or GCP account-level observation from a VM instead of cluster deployment? Use [AWS Installation](/getting-started/installation-aws), [GCP Installation](/getting-started/installation-gcp), or [VM Installation](/getting-started/installation-vm) instead.
</Info>

<Warning>
<strong>Beta Release Notice</strong>

RubixKube is currently in <strong>Beta</strong>. While you can install it on Kubernetes environments for <strong>testing and evaluation</strong>, it is <strong>NOT recommended for production environments</strong> yet.

✅ <strong>Safe for:</strong> Development, staging, testing environments<br/>
❌ <strong>Not ready for:</strong> Production workloads, critical infrastructure<br/>
🔜 <strong>Coming soon:</strong> Production-ready release with SLA guarantees
</Warning>

## Why Install on Kubernetes?

Kubernetes environments give you:
- **Pod-level observability** - See what's happening inside your containers
- **Deployment metrics** - Track replicas, rollouts, and scaling events
- **Namespace isolation** - Organize insights by teams and projects
- **Service discovery** - Monitor inter-service communication and dependencies
- **Event-driven insights** - Get alerts when deployments fail or pods crash

## Prerequisites

### Required Tools

<AccordionGroup>
  <Accordion title="kubectl - Kubernetes CLI" icon="terminal">
    **Installation:**

    ```bash
    # macOS
    brew install kubectl

    # Windows (Chocolatey)
    choco install kubernetes-cli

    # Linux
    curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
    chmod +x kubectl
    sudo mv kubectl /usr/local/bin/
    ```

    **Verify:**
    ```bash
    kubectl version --client
    ```
  </Accordion>

  <Accordion title="Cloud Provider CLI (for cloud clusters)" icon="cloud">
    **For AWS EKS:**
    ```bash
    # macOS
    brew install awscli

    # Windows
    msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2.msi

    # Linux
    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
    unzip awscliv2.zip
    sudo ./aws/install
    ```

    **For Google GKE:**
    ```bash
    # macOS
    brew install google-cloud-sdk

    # Linux: https://cloud.google.com/sdk/docs/install
    ```

    **For Azure AKS:**
    ```bash
    # macOS
    brew install azure-cli

    # Windows
    winget install Microsoft.AzureCLI
    ```
  </Accordion>

  <Accordion title="RubixKube Account" icon="user">
    You need a RubixKube account to get your installation command with API key.

    If you don't have one:
    1. Visit [console.rubixkube.ai](https://console.rubixkube.ai)
    2. [Sign up for free](/getting-started/sign-up)
    3. Verify your email
    4. [Log in](/getting-started/login) to access your dashboard
  </Accordion>
</AccordionGroup>

### Cluster Requirements

<Tabs>
  <Tab title="AWS EKS">
    **Minimum Cluster Specs:**
    - Kubernetes version: 1.24+
    - Node type: t3.medium or larger (2 vCPU, 4GB RAM minimum)
    - Number of nodes: 1+ (2-3 recommended for testing)
    - IAM permissions for kubectl access

    **Recommended for Testing:**
    - Node type: t3.large (2 vCPU, 8GB RAM)
    - Number of nodes: 2-3
    - Enable CloudWatch integration
  </Tab>

  <Tab title="Google GKE">
    **Minimum Cluster Specs:**
    - Kubernetes version: 1.24+
    - Machine type: e2-medium or larger (2 vCPU, 4GB RAM minimum)
    - Number of nodes: 1+ (2-3 recommended for testing)
    - GKE cluster admin permissions

    **Recommended for Testing:**
    - Machine type: e2-standard-2 (2 vCPU, 8GB RAM)
    - Number of nodes: 2-3
    - Enable GKE monitoring
  </Tab>

  <Tab title="Azure AKS">
    **Minimum Cluster Specs:**
    - Kubernetes version: 1.24+
    - VM size: Standard_B2s or larger (2 vCPU, 4GB RAM minimum)
    - Number of nodes: 1+ (2-3 recommended for testing)
    - Azure RBAC permissions

    **Recommended for Testing:**
    - VM size: Standard_D2s_v3 (2 vCPU, 8GB RAM)
    - Number of nodes: 2-3
    - Enable Azure Monitor integration
  </Tab>

  <Tab title="Local KIND">
    **Minimum System Specs:**
    - Docker installed and running
    - 8GB RAM available (4GB minimum)
    - 20GB disk space
    - Linux, macOS, or Windows (with Docker Desktop)

    See [Local KIND Installation](/getting-started/installation-kind) for setup steps.
  </Tab>
</Tabs>

## Step 1: Create or Connect to Your Cluster

<Tabs>
  <Tab title="AWS EKS">
    ### Create an EKS Cluster (if you don't have one)

    **Option 1: Using eksctl (Recommended)**

    ```bash
    # Install eksctl (if not already installed)
    brew install eksctl  # macOS
    # For other OS: https://eksctl.io/installation/

    # Create cluster
    eksctl create cluster \
      --name rubixkube-test \
      --region us-west-2 \
      --nodes 2 \
      --node-type t3.medium \
      --version 1.28
    ```

    This takes **15-20 minutes**.

    **Option 2: Using AWS Console**

    1. Go to [AWS EKS Console](https://console.aws.amazon.com/eks)
    2. Click **"Create cluster"**
    3. Name: `rubixkube-test`
    4. Kubernetes version: 1.24+
    5. Follow wizard to create node groups
    6. Wait for cluster to be active

    ### Connect to Your EKS Cluster

    ```bash
    # Configure kubectl to use your EKS cluster
    aws eks update-kubeconfig --region us-west-2 --name rubixkube-test
    ```

    ### Verify Connection

    ```bash
    kubectl get nodes
    ```
  </Tab>

  <Tab title="Google GKE">
    ### Create a GKE Cluster (if you don't have one)

    **Using gcloud CLI:**

    ```bash
    # Set your project
    gcloud config set project YOUR_PROJECT_ID

    # Create cluster
    gcloud container clusters create rubixkube-test \
      --zone us-central1-a \
      --num-nodes 2 \
      --machine-type e2-medium \
      --cluster-version latest
    ```

    This takes **5-10 minutes**.

    ### Connect to Your GKE Cluster

    ```bash
    # Get credentials for kubectl
    gcloud container clusters get-credentials rubixkube-test --zone us-central1-a
    ```

    ### Verify Connection

    ```bash
    kubectl get nodes
    ```
  </Tab>

  <Tab title="Azure AKS">
    ### Create an AKS Cluster (if you don't have one)

    **Using Azure CLI:**

    ```bash
    # Login to Azure
    az login

    # Create resource group
    az group create --name rubixkube-rg --location eastus

    # Create AKS cluster
    az aks create \
      --resource-group rubixkube-rg \
      --name rubixkube-test \
      --node-count 2 \
      --node-vm-size Standard_B2s \
      --enable-managed-identity \
      --generate-ssh-keys
    ```

    This takes **10-15 minutes**.

    ### Connect to Your AKS Cluster

    ```bash
    # Get credentials for kubectl
    az aks get-credentials --resource-group rubixkube-rg --name rubixkube-test
    ```

    ### Verify Connection

    ```bash
    kubectl get nodes
    ```
  </Tab>

  <Tab title="Local KIND">
    See [Local KIND Installation](/getting-started/installation-kind) for creating a local test cluster.
  </Tab>
</Tabs>

## Step 2: Get Your RubixKube Installation Command

### Access Your Dashboard

1. **Log in** to [console.rubixkube.ai](https://console.rubixkube.ai)
2. After login, you'll see the **"Connect Your First Cluster"** screen
3. **Copy the installation command** - it's pre-configured with your unique API key

The command looks like this:

```bash
kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_UNIQUE_API_KEY"
```

<Warning>
<strong>Important:</strong> Each user has a unique API key. Always copy YOUR command from the dashboard!
</Warning>

---

## Step 3: Install RubixKube Observer

### Run the Installation Command

Paste your command from the dashboard into your terminal:

```bash
kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_API_KEY"
```

**What gets installed:**

```
namespace/rubixkube-system created
serviceaccount/rubixkube-observer created
clusterrole.rbac.authorization.k8s.io/rubixkube-observer created
clusterrolebinding.rbac.authorization.k8s.io/rubixkube-observer created
deployment.apps/rubixkube-observer created
configmap/rubixkube-config created
secret/rubixkube-credentials created
```

## Step 4: Verify Installation

### Check Namespace Created

```bash
kubectl get namespaces | grep rubixkube-system
```

Expected output:
```
rubixkube-system   Active   30s
```

### Check Pod Status

```bash
kubectl get pods -n rubixkube-system
```

**Initial output** (pods starting):
```
NAME                                  READY   STATUS              RESTARTS   AGE
rubixkube-observer-7d4b9c8f6d-abc12   0/1     ContainerCreating   0          15s
```

**Wait 1-2 minutes**, then check again. **Success output:**
```
NAME                                  READY   STATUS    RESTARTS   AGE
rubixkube-observer-7d4b9c8f6d-abc12   1/1     Running   0          2m
```

### Check Observer Logs

```bash
kubectl logs -n rubixkube-system deployment/rubixkube-observer --tail=50
```

**Healthy logs should show:**

```
[INFO] RubixKube Observer v0.x.x starting...
[INFO] Cluster registered: rubixkube-test
[INFO] Connected to RubixKube platform
[INFO] Heartbeat: cluster healthy
```

### Verify in RubixKube Dashboard

1. Go to [console.rubixkube.ai](https://console.rubixkube.ai)
2. Navigate to **Dashboard** or **Environments** page
3. Your cluster should appear with:
   - **Green status indicator** (Connected)
   - **Metrics** starting to populate
   - **Observer Status:** "Active"

<Note>
<strong>Sync Delay:</strong> It may take 1-2 minutes for your cluster to appear in the dashboard.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Pod stuck in ImagePullBackOff">
    **Problem:** Pod can't download the RubixKube image.

    **Solutions:**
    1. Check pod details: `kubectl describe pod -n rubixkube-system -l app=rubixkube-observer`
    2. Verify internet connectivity: `kubectl run test-curl --image=curlimages/curl -i --rm -- curl -I https://api.rubixkube.ai`
    3. If issue persists after 5 minutes, contact support
  </Accordion>

  <Accordion title="Pod in CrashLoopBackOff">
    **Problem:** Pod starts but immediately crashes.

    **Most common cause:** Invalid or missing API key

    **Solution:**
    1. Check logs: `kubectl logs -n rubixkube-system deployment/rubixkube-observer`
    2. Delete and reinstall with correct API key:
    ```bash
    kubectl delete namespace rubixkube-system
    sleep 30
    # Run fresh installation command from dashboard
    ```
  </Accordion>

  <Accordion title="Cluster not appearing in dashboard">
    **Checklist:**
    1. Wait 2-3 minutes for initial sync
    2. Verify pod is running: `kubectl get pods -n rubixkube-system`
    3. Check observer logs for connection errors
    4. Verify API key is correct
    5. Check outbound connectivity to `api.rubixkube.ai:443`
  </Accordion>
</AccordionGroup>

## Managing Your Installation

### Update RubixKube Observer

When a new version is available:

```bash
# Simply rerun the installation command
kubectl apply -f "https://api.rubixkube.ai/install/observer.yaml?apiKey=rk_YOUR_API_KEY"
```

### Restart Observer

```bash
kubectl rollout restart deployment/rubixkube-observer -n rubixkube-system
```

### Check Resource Usage

```bash
kubectl top pod -n rubixkube-system
```

**Typical resource usage:**
- **Memory:** ~255Mi combined
- **CPU:** Under 10 millicores

### Uninstall RubixKube

```bash
# Delete the entire RubixKube namespace
kubectl delete namespace rubixkube-system
```
