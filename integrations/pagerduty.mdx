Connect PagerDuty to page your on-call rotation when a RubixKube insight crosses a severity threshold. Acknowledgement and resolution sync in both directions, so the two systems stay in step.

## What you can do

<CardGroup cols={2}>
  <Card title="Severity-based paging" icon="bell">
    Page on Critical (and optionally High) insights. Lower severities stay in Slack or Teams.
  </Card>
  <Card title="Bidirectional sync" icon="arrows-rotate">
    Acknowledging the page in PagerDuty marks the insight as acknowledged in RubixKube, and vice versa.
  </Card>
  <Card title="Multiple services" icon="layer-group">
    Route different environments or tags to different PagerDuty services for team ownership.
  </Card>
  <Card title="RCA attached" icon="file-magnifying-glass">
    Every page includes a deep link to the RCA report (or the insight, if no RCA has landed yet).
  </Card>
</CardGroup>

## Install

<Steps titleSize="h3">
  <Step title="Create an integration in PagerDuty">
    In PagerDuty, on the target service, create an **Events API v2** integration. Copy the integration key.
  </Step>
  <Step title="Configure in RubixKube">
    In the console, open **Integrations → PagerDuty** and paste the integration key. Pick the environments and severity thresholds that should route to this service.
  </Step>
  <Step title="Test the pipe">
    Click **Send test event**. A low-priority test incident appears in PagerDuty, and the loop confirms in RubixKube.
  </Step>
</Steps>

## Routing rules

The default rule pages on Critical in all production environments. Typical overrides:

- Separate services per team: `payments-oncall`, `platform-oncall`, `data-oncall`.
- Tighter thresholds on regulated environments (page on High too).
- Looser thresholds on lab and dev (no paging, Slack only).

## What a page includes

- Headline from the insight or RCA.
- Severity and affected resources.
- Deep link back to the RubixKube console.
- A one-line root cause summary if an RCA is attached.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Pages are not firing">
    Check the integration key is correct and the Events API integration is enabled on the PagerDuty service. Use **Send test event** to validate end-to-end.
  </Accordion>
  <Accordion title="Pages fire for events you expected to skip">
    Your severity threshold is too low, or the environment mapping is too broad. Tighten the rule in **Integrations → PagerDuty → Rules**.
  </Accordion>
  <Accordion title="Acknowledgement does not sync back">
    Bidirectional sync requires an additional Events API v3 webhook. Follow the prompt in the RubixKube console to add it.
  </Accordion>
</AccordionGroup>
